# First-Match Ablation: Budget vs. Accuracy

## Methodology

Instead of collecting all consistent candidates and voting, we return the answer from the **first** candidate found (across all 3 pipelines). A `_derive_output` call is the atomic unit of work: one digit-tuple × one math operation tried against the encrypted output.

For each problem we record:
- `calls`: number of `_derive_output` calls until the first candidate was found
- `correct`: whether the first candidate's answer matches the expected answer

We then sweep a **budget** (max allowed calls) and report:
- **Solved**: first candidate found within budget AND answer correct
- **Attempted**: first candidate found within budget (right or wrong)
- **Precision**: solved / attempted (answer quality given a match was found)
- **Recall**: solved / total problems
